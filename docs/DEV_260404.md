## 문제 1: MFDS Strategy 3 — 오매칭 버그

### 증상
```
검색어: "제놀 푸로탑"
→ Strategy 3 (base name "제놀") → MFDS hit: "제놀담푸러정(ZD각인)"
→ searchWebTool("제놀담푸러정 성분 용법") 호출  ← 완전히 다른 약 검색!
```

### 원인
`drug-info-tool.ts` L305: Strategy 3은 raw base name(`제놀`)만으로 MFDS 검색,
반환된 item이 원래 검색어(`푸로탑`)을 포함하는지 전혀 검증하지 않음.

### 해결: Strategy 3 제거 + 실패 시 명확한 에러 반환
Strategy 1, 2 모두 실패 → `식약처 DB에 정보 없음` 반환
→ LLM이 `searchWebTool`을 **원래 검색어**("제놀 푸로탑")로 호출하도록 유도

### 수정 파일
- `api/_lib/agent/drug-info-tool.ts`
  - L300-320: Strategy 3 블록 삭제
  - L322: `items.length === 0` 체크 메시지에 searchWebTool 유도 문구 추가

# 에이전트 오케스트레이션 구조 개선 계획 (2026-04-04)

## 현재 구조 (As-Is)

### Intent 분류: 2개
```
Router (Flash-Lite)
  ├─ "general"  → SDK path (@google/genai) → Google Search grounding ✅
  └─ "medical"  → LangChain path           → Google Search grounding ❌
                                            → 커스텀 툴 호출 가능 ✅
```

### 현재 보유 렌더러 7종
| 블록 | 도메인 | 현재 경로 |
|------|--------|-----------|
| `json:drug` | 약품 카드 | medical (LangChain) |
| `json:smiles` | 화학 구조 | general (SDK) |
| `json:bio` | 단백질/DNA 3D | general (SDK) |
| `json:physics` | 물리 시뮬레이션 | general (SDK) |
| `json:diagram` | 경사면 힘 다이어그램 | general (SDK) |
| `json:constellation` | 별자리/천체 | general (SDK) |
| `json:chart` | 통계/차트 | general (SDK) |

### 현재 구조의 핵심 문제
1. **의료 intent 과도하게 뭉뚱그려짐**
   - `"타이레놀 설명해줘"` (약품 카드 필요) vs `"혈압약 먹으면 어지럽나"` (일반 의학 QA)
   - 둘 다 `medical` → LangChain → Google Search 없이 답변
2. **과학 도메인 intent 전혀 없음**
   - `"도파민 구조 보여줘"` → `general` → 우연히 `json:smiles` 출력 (프롬프트 의존)
   - `"오리온자리 별자리 보여줘"` → `general` → 우연히 `json:constellation` 출력
   - 라우터가 도메인을 모르니 모델 선택/프롬프트 최적화 불가
3. **시스템 프롬프트 비효율**
   - 모든 요청에 7개 렌더러 지침 전부 로딩 (~2000 토큰 낭비)
   - 약품 질문인데 `json:constellation` 지침이 같이 로딩됨
4. **LangChain/SDK 분기 구조상 한계**
   - LangChain: 커스텀 툴 ✅ / Google Search grounding ❌
   - SDK: 커스텀 툴 ❌ / Google Search grounding ✅
   - `drug_id`, `drug_info`만 실제로 커스텀 툴 필요 → 나머지 의료/과학은 SDK로 올릴 수 있음

### 원인 흐름
```
[1턴] YouTube URL 포함 → chat.ts에서 transcript fetch 성공
      → webContent에 "[TRANSCRIPT]..." 포함 → LLM 정상 요약
      → App.tsx: lastActiveDoc 업데이트 코드가 video/* mimeType 기준으로만 동작

[2턴] 현재 첨부 없음 + lastActiveDoc 없음 → webContext = ""
      → chat.ts: hasTranscript = false
      → history에서 YouTube URL 감지 → isYoutubeRequest = true + fileData 재첨부
      → 동영상 재분석 (추가 레이턴시 + 토큰 낭비)
```

### 원인 코드


# 시각화 카드 전체화면 팝업 계획 (2026-04-04)

## 기능 개요
시각화 카드(BioRenderer, ChemicalRenderer 등) 우하단에 expand 버튼을 추가하여
클릭 시 모달 팝업으로 해당 시각화를 전체화면 크기로 표시.

## 구현 방식

### 전반 구조
```
카드 우하단 expand 버튼 (fa-expand 아이콘)
    ↓ 클릭
App.tsx expandedViz state 업데이트 → { type, data }
    ↓
createPortal(document.body)로 VisualizationModal 마운트
    - backdrop blur + z-index 최상위
    - 동일 Renderer를 fullscreen 크기로 재렌더링
    - ESC 키 / X 버튼으로 닫기
```

### 상태 구조
```typescript
// App.tsx
const [expandedViz, setExpandedViz] = useState<{
    type: 'bio' | 'chemical' | 'physics' | 'constellation' | 'chart' | 'diagram' | 'drug';
    data: any;
} | null>(null);
```

### 주요 변경 파일
1. `App.tsx` — `expandedViz` state + `VisualizationModal` 컴포넌트 (Portal)
2. `components/ChatMessage.tsx` — 각 Renderer 래퍼에 expand 버튼 + `onExpand` 콜백
3. 각 Renderer 컴포넌트 — `isExpanded?: boolean` prop 수용 (크기 조정용)

### expand 버튼 위치
```tsx
// ChatMessage.tsx 내 각 Renderer 래퍼
<div className="relative group">
    <BioRenderer bioData={part.data} language={language} />
    <button
        onClick={() => onExpand?.({ type: 'bio', data: part.data })}
        className="absolute bottom-2 right-2 opacity-0 group-hover:opacity-100
                   bg-slate-700/80 text-white rounded p-1.5 transition-opacity"
    >
        <i className="fa-solid fa-expand text-xs" />
    </button>
</div>
```

## 렌더러별 고려사항

| 렌더러 | 팝업 적합성 | 특이사항 |
