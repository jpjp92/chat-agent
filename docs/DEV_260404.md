## 문제 1: MFDS Strategy 3 — 오매칭 버그

### 증상
```
검색어: "제놀 푸로탑"
→ Strategy 3 (base name "제놀") → MFDS hit: "제놀담푸러정(ZD각인)"
→ searchWebTool("제놀담푸러정 성분 용법") 호출  ← 완전히 다른 약 검색!
```

### 원인
`drug-info-tool.ts` L305: Strategy 3은 raw base name(`제놀`)만으로 MFDS 검색,
반환된 item이 원래 검색어(`푸로탑`)을 포함하는지 전혀 검증하지 않음.

### 해결: Strategy 3 제거 + 실패 시 명확한 에러 반환
Strategy 1, 2 모두 실패 → `식약처 DB에 정보 없음` 반환
→ LLM이 `searchWebTool`을 **원래 검색어**("제놀 푸로탑")로 호출하도록 유도

### 수정 파일
- `api/_lib/agent/drug-info-tool.ts`
  - L300-320: Strategy 3 블록 삭제
  - L322: `items.length === 0` 체크 메시지에 searchWebTool 유도 문구 추가

## 문제 2: ConnectDI URL — 구형 한글 단위 미처리

### 증상
- MFDS 구형 표기: `타이레놀정500밀리그람(아세트아미노펜)` (`밀리그람`)
- `ChatMessage.tsx` regex: `밀리그램`만 처리, `밀리그람` 미처리
- → ConnectDI URL에 `밀리그람`이 그대로 들어가 검색 실패

### 해결
`밀리그[램람]`, `마이크로그[램람]`, `그[램람]` 패턴으로 확장

### 수정 파일
- `components/ChatMessage.tsx`
  - L661: `normName` 계산 regex 수정

## 문제 3: searchWebTool — 레퍼런스 소스 칩 미반환

### 현재 동작
- `searchWebTool`: DDG 텍스트 스니펫만 반환, URL 미파싱
- 폴백 검색 시 하단 소스 칩 없음

### 해결
DDG `/lite/` 엔드포인트에서 URL도 파싱:
```
result__url 클래스 → href 추출 → [WEB_SOURCE_URLS] 블록으로 반환
```
`chat.ts`에서 tool 응답 내 `[WEB_SOURCE_URLS]` 파싱 → `allSources`에 추가 → SSE `sources` 이벤트 발송

### 수정 파일
1. `api/_lib/agent/tools.ts`
   - `searchWebTool`: URL 파싱 및 `[WEB_SOURCE_URLS]` 블록 추가
2. `api/chat.ts`
   - tool 결과 처리 부분에 `[WEB_SOURCE_URLS]` 파싱 로직 추가
