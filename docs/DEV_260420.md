# DEV_260420 — URL 요약 시 [1] 인용 마커 제거

## 문제

URL 기반 웹페이지 요약 시 응답 뒤에 `[1]`, `[2]` 같은 인용 마커가 반복 출력됨.

## 원인 분석

### 원인 1: source text echo (주요 원인)
- `fetch-url.ts`에서 HTML을 단순 태그 제거로 파싱 (`replace(/<[^>]+>/g, ' ')`)
- 원본 페이지의 각주 마커 `[1]`, `[2]`가 raw text로 살아남아 `[PROVIDED_SOURCE_TEXT]`에 포함
- Gemini가 source text의 `[1]`을 요약문에 echo하거나 참조

### 원인 2: Google Search grounding fallback
- URL fetch 실패 시 `[URL_CONTENT:]` 태그 없음 → `hasUrlContent = false`
- `generator.ts`에서 `useGoogleSearch = true` 유지 → Google Search grounding 활성화
- Gemini가 검색 결과 인용으로 `[1]`, `[2]` 자동 삽입 (이건 정상 동작)

## 수정 내용 ✅

### fix 1: fetch-url.ts — 전처리 (원인 1 차단)
- 본문 텍스트 클리닝 단계에서 각주 마커 제거: `.replace(/\[\d+\]/g, '')`
- 본문 크기 18000 → 12000자, 최종 반환 20000 → 14000자로 축소
  - 이유: 긴 블로그(Medium 등) 요약 시 Vercel 무료플랜 60초 제한 초과 방지
  - YouTube는 별도 분기라 영향 없음 / PDF 웹링크는 attachment로 직접 전달되어 영향 없음

### fix 2: generator.ts — 조건부 후처리 (원인 1 잔여분 + 안전망)
- `useGoogleSearch === false`인 경우에만 응답에서 `[N]` 제거
- Google Search가 켜진 정상 grounding 인용은 그대로 유지

## 변경 파일
- `api/fetch-url.ts`
- `api/_lib/agent/nodes/generator.ts`
