## 발견된 문제 목록

### 🔴 P1 — `geminiService.ts` fetch 함수 `response.ok` 미체크

**대상 함수 (6개)**
- `loginUser` / `updateRemoteUserProfile`
- `fetchSessions` / `createSession` / `fetchSessionMessages`
- `deleteSession` / `updateSessionTitle`

**현재 코드 패턴**
```typescript
export const fetchSessions = async (userId: number) => {
  const response = await fetch(`/api/sessions?user_id=${userId}`);
  return response.json(); // ← HTTP 4xx/5xx도 그대로 파싱
};
```

**문제**
- 서버 500/502 시 HTML 에러 페이지가 `response.json()` 파싱에서 SyntaxError throw
- 또는 `{ error: "..." }` JSON이 반환되어도 상위 코드가 `sessions.length` 접근 시 TypeError
- silent failure 후 원인 파악 불가

**수정 방향**
```typescript
export const fetchSessions = async (userId: number) => {
  const response = await fetch(`/api/sessions?user_id=${userId}`);
  if (!response.ok) throw new Error(`fetchSessions failed: ${response.status}`);
  return response.json();
};
```
또는 DEV_260405에서 검토된 Result 패턴 도입 시 일괄 적용.

### 🔴 P2 — `useChatStream.ts` 스트리밍 에러 catch에서 `onError()` 미호출

**현재 코드**
```typescript
// src/hooks/useChatStream.ts
} catch (error: any) {
```

**문제**
- 스트리밍 오류 시 로딩 상태 표시줄에 에러 메시지가 잠깐 표시되다 사라짐

**수정 방향**
```typescript
} catch (error: any) {
```

### 🟡 P4 — `geminiService.ts` SSE 라인 JSON.parse 무방어

**현재 코드**
```typescript
for (const line of lines) {
  if (line.startsWith('data: ')) {
    const data = JSON.parse(line.slice(6));  // ← malformed 시 throw → 전체 스트림 catch로
    if (data.error) throw new Error(data.error);
    if (data.text) onChunk(data.text, false);
    if (data.sources && onMetadata) onMetadata(data.sources);
  }
}
```

**문제**
- SSE 라인이 버퍼 경계에서 잘린 경우 (`data: {"text": "안녕`) 파싱 실패
- 개별 라인 파싱 오류가 전체 스트리밍 세션을 종료시킴

**수정 방향**
```typescript
for (const line of lines) {
  if (line.startsWith('data: ')) {
    try {
      const data = JSON.parse(line.slice(6));
      if (data.error) throw new Error(data.error);
      if (data.text) onChunk(data.text, false);
      if (data.sources && onMetadata) onMetadata(data.sources);
    } catch (e) {
      // 개별 라인 파싱 실패는 스킵, 스트림 유지
      if (e instanceof SyntaxError) continue;
      throw e; // data.error로 생성된 에러는 상위로 전파
    }
  }
}
```

### `useChatSessions.ts` — `mapDbMessage` attachment 복원 불완전

```typescript
const mapDbMessage = (message: any): Message => ({
  // ...
  attachment: message.attachment_url
    ? {
        fileName: message.attachment_url.includes('pdf') ? 'document.pdf' : 'image_attached',

      }
    : undefined,

```

세션 히스토리 로드 시 첨부 파일 메타데이터가 올바르게 복원되지 않음. `attachment_url`이 mimeType 자리에 들어가 있어 컴포넌트에서 mimeType 판단 오류 가능.

### `useChatStream.ts` — URL 처리 블록 에러 전파 경로 불명확

URL fetch / YouTube transcript 처리가 `try-catch` 없이 메인 흐름에 위치하여,
내부 함수가 예외를 throw하면 외부의 `streamChatResponse` try-catch로 올라가 에러 원인 특정이 어려움.
URL 처리 구간을 별도 try-catch로 감싸 "URL 콘텐츠 가져오기 실패" 수준의 명시적 에러 메시지 제공 권장.

## 의약품 이미지 간헐적 미표시 버그 수정 (2026-04-07)

### 증상
의약품 검색 카드에서 이미지가 간헐적으로 표시되지 않거나, 직전에 표시되던 이미지가 사라지는 현상.

### 원인 분석

#### 1. Race condition — `DrugRenderer.tsx`
`useEffect` cleanup이 없어 이전 drug의 `syncImage()` 요청이 새로운 drug 렌더 이후에도 계속 실행되다가 완료 시 state를 덮어쓰는 문제.

- 시나리오: Drug A → Drug B 전환 중, Drug A 요청 실패 → `setImageError(true)` 호출 → Drug B 카드에 "이미지 준비 중" 표시
- `setSyncing(true)` 초기화 후 이전 요청의 `finally { setSyncing(false) }`가 뒤늦게 실행되어 로딩 상태가 즉시 종료되는 문제도 동반

#### 2. 스코프 버그 — `sync-drug-image.ts`
`fileName`, `resolveInflight` 변수가 `try` 블록 안에서 `const`/`let`으로 선언되어 `catch` 블록에서 접근 불가.
예상치 못한 예외 발생 시 inflight Promise가 `resolve`되지 않아 동일 drug의 후속 요청이 무한 대기.

### 수정 내용

**`components/DrugRenderer.tsx`**
- `AbortController` 추가 및 cleanup 함수에서 `controller.abort()` 호출
- `fetch` 요청에 `signal: controller.signal` 전달
- catch에서 `AbortError`는 state 변경 없이 조용히 무시

```typescript
const controller = new AbortController();
const syncImage = async () => {
    try {
        const response = await fetch('/api/sync-drug-image', {
            ...
            signal: controller.signal
        });
        ...
    } catch (error: any) {
        if (error.name === 'AbortError') return;  // 이전 요청 취소는 무시
        setImageError(true);
    } finally {
        setSyncing(false);
    }
};
syncImage();
return () => controller.abort();  // cleanup
```

**`api/sync-drug-image.ts`**
- `fileName`, `resolveInflight` 선언을 `try` 블록 **바깥**으로 이동
- `catch`에서 `resolveInflight?.(null)` / `inflightRequests.delete(fileName)` 안전하게 호출 가능

```typescript
// try 블록 밖에서 선언
let fileName: string | undefined;
let resolveInflight: ((v: ...) => void) | undefined;

try {
    ...
    fileName = `drug-cache/${urlHash}.jpg`;
    inflightRequests.set(fileName, new Promise(r => { resolveInflight = r; }));
    ...
} catch (error: any) {
    resolveInflight?.(null);              // 이제 정상 접근 가능
    inflightRequests.delete(fileName);    // 이제 정상 접근 가능
    ...
}
```
