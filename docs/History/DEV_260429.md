# maxOutputTokens 인텐트별 최적화 (2026-04-29 — 완료)

## 배경

Vercel Hobby 플랜은 `maxDuration` 상한이 **60초 고정**이라 vercel.json 수정으로 타임아웃 연장 불가.  
이미지 분석 응답 잘림 문제(DEV_260427 참고)의 실질적 해결책은 **생성 시간 자체를 줄이는 것**.

## 최종 분기 로직 (`generator.ts`)

```ts
const resolvedMaxTokens = (() => {
    if (hasDocumentContent) return 16384;               // PDF·문서 분석
    if (isYoutubeRequest) return 8192;                  // YouTube 요약 (transcript + native video 모두)
    if (hasMultimodalContent) return 4096;              // 이미지 분석
    if (hasUrlContent) return 8192;                     // URL 요약
    if (state.intent === 'data_viz') return 8192;       // 차트 JSON
    return 32768;                                        // 코드·일반
})();
```

- `hasDocumentContent`: PDF mimeType 감지 플래그. `fileData` 분기와 `image_url` 분기 양쪽에서 세팅.
- `hasDocumentContent`가 `hasMultimodalContent`보다 앞에 위치 — PDF 요청은 인텐트 무관하게 16384 적용.
- `isYoutubeRequest`가 `hasMultimodalContent`보다 앞에 위치 — 트랜스크립트 없이 native video(`fileData: video/mp4`) fallback 시 `hasMultimodalContent=true`가 되어 4096으로 잘못 분기되던 문제 수정.

## 실측 검증 결과

| 케이스 | maxTokens | responseLen | 결과 |
|---|---|---|---|
| 이미지 분석 (`general`) | 4,096 | 2,080자 | 정상 완료 ✓ |
| 코드 리뷰 (`general`) | 32,768 | 5,332자 | 정상 완료 ✓ |
| PDF 분석 (`data_viz`) | 16,384 | 1,811자 | 정상 완료 ✓ (이전 MAX_TOKENS 잘림 해소) |
| YouTube 요약 (transcript 경로) | 8,192 | 2,799자 | 정상 완료 ✓ |
| YouTube 요약 (native video 경로) | 8,192 | 2,249자 | 정상 완료 ✓ (이전 4096 오분기 수정) |

**PDF 수정 경위**:
- 1차: `image_url` 분기에만 `hasDocumentContent = true` 추가 → PDF는 `fileData` 분기를 타므로 무효 (maxTokens: 4096 그대로)
- 2차: `fileData` 분기에도 `if (part.fileData.mimeType === 'application/pdf') hasDocumentContent = true` 추가 → maxTokens: 16384 적용 확인

**확인된 동작**: PDF + `data_viz` 인텐트 조합에서 `hasDocumentContent`가 우선 적용되어 16384 사용.  
`data_viz`(8192)로 줄어들지 않음 — 조건 순서 설계가 의도대로 동작.

## 변경 파일 목록

| 파일 | 변경 내용 |
|---|---|
| `api/_lib/agent/nodes/generator.ts` | `hasDocumentContent` 플래그 추가, `resolvedMaxTokens` IIFE, `maxOutputTokens` 2곳 교체, catch 블록 `cutOff` 이벤트 |
| `api/chat.ts` | streamEvents 루프 완료 후 `sendEvent({ done: true })` 추가 |
| `services/geminiService.ts` | `onCutOff?` 파라미터, `data.cutOff` / `data.done` SSE 파싱 |
| `types.ts` | `Message` 인터페이스에 `isCutOff?: boolean` 추가 |
| `src/hooks/useChatStream.ts` | `onCutOff` 콜백 전달, 메시지 `isCutOff: true` 반영 |
| `components/ChatMessage.tsx` | `isCutOff` amber 경고 배너 렌더링 |

# YouTube 요약 최적화 (2026-04-29 — 완료)

## 배경

YouTube 요약 시 중간 끊김 및 응답 지연 문제 발생. 원인 분석:

1. **트랜스크립트 길이 무제한**: 1시간짜리 영상 = ~48,000자 → 입력 토큰 폭증 → Gemini 추론 지연 + 60초 초과 위험
2. **native video 경로에서 maxTokens 오분기**: 트랜스크립트 없을 때 `fileData: video/mp4` → `hasMultimodalContent=true` → 이미지 분기(4096) 적용되던 버그
3. **YouTube URL 칩 중복**: `manualGroundingSources`(사용자 입력 URL 그대로)와 `pendingSources`(backend 정규화 URL)의 URI가 달라 Map dedup 실패 → 두 개 렌더링
4. **fetch-transcript 페이지 fetch timeout 과도**: 25초 설정 → 자막 없는 영상 대기 과도

## 수정 내용

### 1. 트랜스크립트 20,000자 상한 (`useChatStream.ts`)

```ts
const MAX_TRANSCRIPT_CHARS = 20000;
const trimmedTranscript = transcript.length > MAX_TRANSCRIPT_CHARS
  ? transcript.slice(0, MAX_TRANSCRIPT_CHARS) + '\n\n[자막이 너무 길어 일부만 제공됩니다]'
  : transcript;
webContext += `\n\n${metadata}\n\n[TRANSCRIPT]\n${trimmedTranscript}`;
```

20,000자 ≈ 약 25분 분량 영상. 초과 시 모델에게 힌트 전달하여 불완전 컨텍스트를 인식하게 함.

### 2. `isYoutubeRequest` 체크 순서 변경 (`generator.ts`)

`hasMultimodalContent`보다 앞으로 이동 → native video fallback 경로도 8192 적용.

### 3. YouTube maxOutputTokens 12,288 → 8,192 (`generator.ts`)

YouTube 요약 응답은 실측 ~2,500~3,000자 수준 — 12,288은 과도했음.

### 4. YouTube URL 정규화 (`useChatStream.ts`)

```ts
const isYt = cleanUrl.includes('youtube.com') || cleanUrl.includes('youtu.be');
let normalizedUrl = cleanUrl;
if (isYt) {
  const ytIdMatch = cleanUrl.match(/(?:v=|youtu\.be\/|shorts\/)([a-zA-Z0-9_-]{11})/);
  if (ytIdMatch) normalizedUrl = `https://www.youtube.com/watch?v=${ytIdMatch[1]}`;
}
```

`youtu.be/ID`, `shorts/ID` 등 다양한 형식 → `youtube.com/watch?v=ID` 정규화. backend `pendingSources`와 동일 URI → dedup 정상 동작.

### 5. 페이지 fetch timeout 단축 (`fetch-transcript.ts`)

25,000ms → 12,000ms. 자막 없는 영상의 대기 시간 최대 13초 단축.

## 변경 파일 목록

| 파일 | 변경 내용 |
|---|---|
| `api/_lib/agent/nodes/generator.ts` | `isYoutubeRequest` 조건 `hasMultimodalContent` 앞으로 이동, YouTube maxTokens 12288 → 8192 |
| `api/fetch-transcript.ts` | 페이지 fetch timeout 25000ms → 12000ms |
| `src/hooks/useChatStream.ts` | 트랜스크립트 20,000자 상한, YouTube URL 정규화 (dedup 수정) |

# 모바일 SSE 연결 안정성 강화 (2026-04-29 — 완료)

## 배경

모바일에서 일반보다 에러가 자주 발생. 원인: SSE 스트림에 AbortController·타임아웃이 없어서 연결이 조용히 끊겨도 클라이언트가 감지 불가 → `reader.read()` 무한 대기.

## 수정 내용

### 1. Activity Monitor (`geminiService.ts`)
`AbortController` + 5초 인터벌로 마지막 SSE 수신 시각을 체크. **25초**(heartbeat 8s × 3회 미수신) 무활동이면 연결 강제 종료 → `AbortError` → "응답을 받지 못했습니다" 재시도 가능 에러로 변환.

### 2. 부분 응답 드롭 감지 (`geminiService.ts`)
`receivedDone` 플래그 추가. 텍스트는 수신했지만 `done: true` 이벤트 없이 스트림이 닫히면 → `onCutOff()` 호출 → amber 경고 배너.

### 3. 재시도 조건 확장 (`useChatStream.ts`)
`Failed to fetch`, `TypeError`(네트워크 오류)도 재시도 대상에 추가 → 모바일 요청 자체 실패 시 자동 1회 재시도.

### 4. MAX_TOKENS cutOff 이벤트 누락 수정 (`generator.ts`)
`finishReason === 'MAX_TOKENS'` 감지 시 로그만 찍고 `cutOff` 이벤트를 보내지 않던 버그. `hitMaxTokens` 플래그 추가 → 스트림 종료 후 `sendEvent({ cutOff: true })`.

## 변경 파일 목록

| 파일 | 변경 내용 |
|---|---|
| `services/geminiService.ts` | AbortController + activityMonitor(25s), `receivedDone` 플래그, AbortError → 재시도 에러 변환 |
| `src/hooks/useChatStream.ts` | `isRetryable`에 `Failed to fetch` / `TypeError` 추가 |
| `api/_lib/agent/nodes/generator.ts` | `hitMaxTokens` 플래그, MAX_TOKENS 시 `sendEvent({ cutOff: true })` |

# YouTube 멀티턴 미리보기 재등장 버그 수정 (2026-04-29 — 완료)

## 배경

YouTube 영상을 1턴에서 요약 후, 2턴에서 세부 내용을 질의하면 AI 응답 메시지에 YouTube 미리보기(임베드)가 다시 렌더링되는 문제.

## 원인 분석

**버그 1 — groundingSources 무조건 전송** (`chat.ts`):
`isYoutubeRequest = true`이면 멀티턴 follow-up에서도 YouTube URL을 `groundingSources`로 전송 → `ChatMessage.tsx`의 `<YoutubeEmbed>` 렌더링 조건 충족 → 임베드 재등장.

**버그 2 — native 영상 재전송 가능성** (`chat.ts`):
`hasVideoSummary` 하나에만 의존. `fetchUrlContent` 실패 등으로 `lastActiveDoc` 미저장 시 `hasVideoSummary = false` → 2턴에서도 `fileData: video/mp4` 재전송 → Gemini가 영상 재분석.

## 수정 내용 (`chat.ts`)

`isYoutubeFromPrompt` 플래그 도입 — 현재 프롬프트에 YouTube URL이 있는 신규 분석 요청인지 여부를 명시적으로 구분:

- **native 영상 전송**: `isYoutubeFromPrompt && !hasTranscript && !hasVideoSummary` 조건으로 변경
- **groundingSources 전송**: `isYoutubeFromPrompt`일 때만 → 2턴 임베드 재렌더링 차단

## 검증

1턴 영상 요약 후 2턴 세부 질의 테스트 — YouTube 미리보기 미등장, 이전 타임스탬프 링크 정상 참조 확인.

## 변경 파일 목록

| 파일 | 변경 내용 |
|---|---|
| `api/chat.ts` | `isYoutubeFromPrompt` 플래그, native 영상 전송 조건 변경, groundingSources 전송 조건 변경 |
