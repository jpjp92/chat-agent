## 발견된 문제 목록

### 🔴 P1 — `geminiService.ts` fetch 함수 `response.ok` 미체크

**대상 함수 (6개)**
- `loginUser` / `updateRemoteUserProfile`
- `fetchSessions` / `createSession` / `fetchSessionMessages`
- `deleteSession` / `updateSessionTitle`

**현재 코드 패턴**
```typescript
export const fetchSessions = async (userId: number) => {
  const response = await fetch(`/api/sessions?user_id=${userId}`);
  return response.json(); // ← HTTP 4xx/5xx도 그대로 파싱
};
```

**문제**
- 서버 500/502 시 HTML 에러 페이지가 `response.json()` 파싱에서 SyntaxError throw
- 또는 `{ error: "..." }` JSON이 반환되어도 상위 코드가 `sessions.length` 접근 시 TypeError
- silent failure 후 원인 파악 불가

**수정 방향**
```typescript
export const fetchSessions = async (userId: number) => {
  const response = await fetch(`/api/sessions?user_id=${userId}`);
  if (!response.ok) throw new Error(`fetchSessions failed: ${response.status}`);
  return response.json();
};
```
또는 DEV_260405에서 검토된 Result 패턴 도입 시 일괄 적용.

### 🔴 P2 — `useChatStream.ts` 스트리밍 에러 catch에서 `onError()` 미호출

**현재 코드**
```typescript
// src/hooks/useChatStream.ts
} catch (error: any) {
  hasError = true;
  setLoadingStatus(error.message);          // ← 로딩 UI 영역에 에러 텍스트 5초
  setTimeout(() => setLoadingStatus(null), 5000);
  // onError() 미호출 → Toast 없음
}
```

**문제**
- 스트리밍 오류 시 로딩 상태 표시줄에 에러 메시지가 잠깐 표시되다 사라짐
- Toast가 뜨지 않아 사용자가 오류를 인지하기 어려움
- `loadingStatus`가 에러와 상태 두 가지 역할을 겸함 → 의미 혼재

**수정 방향**
```typescript
} catch (error: any) {
  hasError = true;
  onError(error.message);  // Toast 표시
} finally {
  setIsTyping(false);
  setLoadingStatus(null);
  setEditingMessageContent(undefined);
}
```

### 🟡 P4 — `geminiService.ts` SSE 라인 JSON.parse 무방어

**현재 코드**
```typescript
for (const line of lines) {
  if (line.startsWith('data: ')) {
    const data = JSON.parse(line.slice(6));  // ← malformed 시 throw → 전체 스트림 catch로
    if (data.error) throw new Error(data.error);
    if (data.text) onChunk(data.text, false);
    if (data.sources && onMetadata) onMetadata(data.sources);
  }
}
```

**문제**
- SSE 라인이 버퍼 경계에서 잘린 경우 (`data: {"text": "안녕`) 파싱 실패
- 개별 라인 파싱 오류가 전체 스트리밍 세션을 종료시킴

**수정 방향**
```typescript
for (const line of lines) {
  if (line.startsWith('data: ')) {
    try {
      const data = JSON.parse(line.slice(6));
      if (data.error) throw new Error(data.error);
      if (data.text) onChunk(data.text, false);
      if (data.sources && onMetadata) onMetadata(data.sources);
    } catch (e) {
      // 개별 라인 파싱 실패는 스킵, 스트림 유지
      if (e instanceof SyntaxError) continue;
      throw e; // data.error로 생성된 에러는 상위로 전파
    }
  }
}
```

### 🟡 P5 — `useChatSessions.ts` `fetchSessions` error 필드 무시

**현재 코드**
```typescript
const { sessions: dbSessions } = await fetchSessions(resolvedUserId);
// ← error 필드 무시

if (dbSessions && dbSessions.length > 0) {
  // ...
}
await createNewSession(resolvedUserId); // ← 세션 로드 실패해도 조용히 신규 생성
```

**문제**
- 서버 에러로 세션 목록 로드 실패 시 빈 배열로 처리되어 새 세션이 생성됨
- 사용자가 기존 대화 내역이 사라진 것으로 오인 가능
- 콘솔 에러도 없어 디버깅 어려움

**수정 방향**
```typescript
const { sessions: dbSessions, error } = await fetchSessions(resolvedUserId);
if (error) {
  reportError(new Error(error), 'Failed to load sessions');
  return; // 실패 시 새 세션 생성하지 않음
}
```

URL fetch / YouTube transcript 처리가 `try-catch` 없이 메인 흐름에 위치하여, 에러 발생 시 로딩 스피너 영구 고착 및 원인 특정 불가.
