# DEV_260420 — URL 요약 개선 + 다크모드 개선 + UI 버그 수정

## 문제

URL 기반 웹페이지 요약 시 응답 뒤에 `[1]`, `[2]` 같은 인용 마커가 반복 출력됨.

## 원인 분석

### 원인 1: source text echo (주요 원인)
- `fetch-url.ts`에서 HTML을 단순 태그 제거로 파싱 (`replace(/<[^>]+>/g, ' ')`)
- 원본 페이지의 각주 마커 `[1]`, `[2]`가 raw text로 살아남아 `[PROVIDED_SOURCE_TEXT]`에 포함
- Gemini가 source text의 `[1]`을 요약문에 echo하거나 참조

### 원인 2: Google Search grounding fallback
- URL fetch 실패 시 `[URL_CONTENT:]` 태그 없음 → `hasUrlContent = false`
- `generator.ts`에서 `useGoogleSearch = true` 유지 → Google Search grounding 활성화
- Gemini가 검색 결과 인용으로 `[1]`, `[2]` 자동 삽입 (이건 정상 동작)

## 수정 내용 ✅

### fix 1: fetch-url.ts — 전처리 (원인 1 차단)
- 본문 텍스트 클리닝 단계에서 각주 마커 제거: `.replace(/\[\d+\]/g, '')`
- 본문 크기 18000 → 12000자, 최종 반환 20000 → 14000자로 축소
  - 이유: 긴 블로그(Medium 등) 요약 시 Vercel 무료플랜 60초 제한 초과 방지
  - YouTube는 별도 분기라 영향 없음 / PDF 웹링크는 attachment로 직접 전달되어 영향 없음

### fix 2: generator.ts — 조건부 후처리 (원인 1 잔여분 + 안전망)
- `useGoogleSearch === false`인 경우에만 응답에서 `[N]` 제거
- Google Search가 켜진 정상 grounding 인용은 그대로 유지

## fix 3: 다크모드 개선 — 미드나잇 인디고 B+1 스타일 적용
- 배경 그라디언트: `#0f1117` 계열 → `#141629 ~ #1a1f3c` (인디고 네이비)
- Ambient orb: `indigo-900/30 + blue-900/20 + purple-900/20` → `#6366f1` 25% + `#a78bfa` 15% (2개로 단순화)
- 헤더/사이드바/인풋바 glass: `slate-800/60` → `white/7%`, 보더 `white/10` → `white/12~13%`
- 유저 채팅 버블: `#2a2d3e` → `indigo-500/20` + `indigo-500/30` 보더 + `indigo-100` 텍스트
- blockquote 다크모드 스타일: `border-white/10` → `border-indigo-400/50`, 텍스트 `text-white/70` 추가 (새 배경에서 안 보이던 문제 수정)
- `LoadingScreen.tsx` 배경도 동일하게 통일

## fix 4: URL 요약 한 줄 요약 blockquote 스타일 복원
- `prompt.ts`: `**한 줄 요약**` + `> (내용)` 구조 → `> **한 줄 요약**` + `> (내용)` 으로 수정
- 헤딩과 텍스트가 같은 blockquote 블록 안에 묶여서 렌더링되도록 수정

## fix 5: 모바일 사이드바 새 채팅 폰트 크기 불일치
- 원인: `index.css`에서 모바일 input에 `font-size: 16px !important` 강제 적용 (iOS 줌 방지)
  → 검색창은 16px, 새 채팅 버튼은 `text-[13px]` 그대로라 크기 차이 발생
- 수정: `ChatSidebar.tsx` 새 채팅 버튼 텍스트에 `text-[16px] sm:text-[13px]` 적용

## fix 6: PDF URL 요약 포맷 통일
- `[URL_PDF_LINK_QUEUED]`, `[ARXIV_PDF_LINK_QUEUED]` 태그에 대한 프롬프트 지침 추가
- 기존엔 PDF URL 요약 시 모델이 자유 형식으로 응답 → 일반 URL 요약과 포맷 불일치
- 이제 `[URL_CONTENT]`와 동일한 `> **한 줄 요약**` + 주요 내용 + 핵심 포인트 구조 적용

## fix 7: 날씨 테이블 이모지 누락 — 맑음
- 원인: 프롬프트 이모지 가이드(`🌞 맑음`)와 예시 테이블(`🌤️ 맑음`) 불일치 → 모델이 혼란으로 이모지 생략
- 예시 테이블을 `🌞 맑음`으로 가이드와 통일
- "이모지 없이 조건만 쓰지 말 것" CRITICAL 지침 추가

## fix 8: 프롬프트 URL 요약 헤딩 다국어 대응
- `getSystemInstruction` 을 arrow 함수 → block 함수로 변환, `URL_SUMMARY_LABELS` 맵 추가
- 언어별 헤딩 주입: Korean(한 줄 요약/주요 내용/핵심 포인트), English, Spanish, French
- 기존엔 `ENTIRE RESPONSE IN ENGLISH` 지시에도 한국어 헤딩이 고정 출력되는 충돌 발생

## fix 9: 소스 텍스트 크기 재조정 (12000 → 15000자)
- 12000자 축소 이후 긴 기술 아티클 요약 품질 저하 확인
- 후반부 핵심 논지가 잘리는 문제 → 15000자(최종 17000자)로 타협점 조정
- Vercel 무료 60초 제한 내에서 품질과 속도 균형

## fix 10: maxOutputTokens 복원 (16384 → 8192)
- 이전 커밋(`3bb15750`) 비교 분석 결과, fix 세션 중 16384로 2배 증가된 것이 속도 저하 + Vercel 60초 타임아웃 원인으로 확인
- generator.ts 3군데(SDK 스트리밍, SDK fallback, LangChain) 모두 8192로 복원
- 기대 효과: 응답 속도 회복, PDF/장문 아티클 요약 타임아웃 감소

## fix 11: 한 줄 요약 blockquote CRITICAL 규칙 추가
- `prompt.ts`의 `[URL_CONTENT]`, `[URL_PDF_LINK_QUEUED]`/`[ARXIV_PDF_LINK_QUEUED]` 섹션 양쪽에 CRITICAL 지침 추가
- 모델이 `> **한 줄 요약**` 두 줄에서 `"> "` 마커를 누락하는 현상 방지
- "Output them as Markdown blockquote lines — do NOT omit the "> " prefix under any circumstance"

## fix 12: JS 렌더링 사이트 Google Search fallback
- `generator.ts` `hasUrlContent` 판정에 최소 길이 기준 300자 추가
- JS 렌더링 의존 사이트는 서버사이드 fetch로 스켈레톤 HTML만 반환 → 본문이 짧아도 `[URL_CONTENT]` 태그가 붙어 Google Search가 비활성화되던 문제
- 300자 미만이면 fetch 실패로 간주하여 Google Search fallback 허용

## fix 13: 한 줄 요약 blockquote 지침 정리
- `prompt.ts` URL_CONTENT / PDF 포맷 블록 내 CRITICAL 주석을 헤더 괄호 설명으로 통합
- CRITICAL 텍스트가 포맷 블록 안에 있어 모델이 포맷 지침과 혼동하던 문제 해결
- `(the first two lines MUST be output as Markdown blockquotes starting with "> ")` 형태로 정리

## fix 14: 504 HTML 응답 crash 방지
- `geminiService.ts` `!response.ok` 처리 개선
- Vercel 504 타임아웃 등에서 HTML 응답 반환 시 `response.json()` → `SyntaxError` crash 발생하던 문제
- content-type 헤더 확인 후 JSON일 때만 파싱, 아니면 status code 기반 메시지 반환

## fix 15: 재시도 딜레이 단축 (1500ms → 500ms)
- `useChatStream.ts` 스트리밍 실패 재시도 대기 시간 단축
- 사용자 체감 응답 속도 개선
