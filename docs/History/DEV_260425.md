## 기존 로직 영향 검토

### Vercel 함수 타임아웃 — 주의 필요

`vercel.json` `maxDuration: 60` 설정 기준, Gemini 2.5 Flash ~200토큰/초 가정:

| 응답 길이     | 생성 시간 | 60s 타임아웃      |
| ------------- | --------- | ----------------- |
| 8,192 tokens  | ~41초     | 안전 ✅           |
| 16,000 tokens | ~80초     | **초과 ❌** |
| 32,768 tokens | ~164초    | **초과 ❌** |

16,000토큰(~13,000자) 이상 응답은 스트림 도중 강제 종료 가능.
일반적인 응답 범위(~5,000토큰)에서는 문제 없음.
코드 대량 생성 등 극단적 케이스에서만 해당. `maxDuration` 상향 시 완전 해소.

### 프론트엔드 — 영향 없음

- `setSessions` 청크별 호출 구조는 기존과 동일, 토큰 수 증가가 빈도에 영향 없음
- `react-markdown` 렌더링: 매우 긴 응답(30,000토큰+)에서 느려질 수 있으나 실용 범위 밖
- Supabase `TEXT` 타입 제한 없음 — DB 저장 영향 없음

### LangChain 경로 (drug_id, drug_info) — 영향 없음

약품 응답은 원래 짧아 실질적 변화 없음.

### 3. 보안 헤더 추가 (`vercel.json`)

CSP는 KaTeX·Google Fonts·FontAwesome·cdnjs 등 외부 리소스 및 inline style 의존성으로 인해 **보류**. 리스크 없는 4종만 적용.

| 헤더                          | 값                                           | 효과                     |
| ----------------------------- | -------------------------------------------- | ------------------------ |
| `X-Content-Type-Options`    | `nosniff`                                  | MIME 스니핑 방지         |
| `X-Frame-Options`           | `SAMEORIGIN`                               | 클릭재킹 방지            |
| `Referrer-Policy`           | `strict-origin-when-cross-origin`          | 리퍼러 최소화            |
| `Strict-Transport-Security` | `max-age=63072000; includeSubDomains`      | HTTPS 2년 강제           |
| `Permissions-Policy`        | `camera=(), microphone=(), geolocation=()` | 불필요 브라우저 API 차단 |

**CSP 미적용 사유**: `fonts.googleapis.com`, `fonts.gstatic.com`, `cdnjs.cloudflare.com`, `cdn.jsdelivr.net` 6개 외부 출처 + KaTeX·react-markdown inline style → `'unsafe-inline'` 허용 시 XSS 방어 효과 반감, 미허용 시 수식·폰트·아이콘 무음 장애 위험.

### 4. SSRF 사설IP 차단 (`fetch-url.ts`, `proxy-image.ts`)

hostname 기반 차단. redirect 우회는 알려진 한계로 감수.

**차단 대상**: `localhost`, `127.x.x.x`, `0.0.0.0`, `169.254.x.x`(AWS/GCP 메타데이터), `::1`

**`fetch-url.ts`**: 기존 `try` 블록 앞에 선행 배치. `new URL()` 파싱 실패 시 `Invalid URL` 400 자동 반환.

```ts
try {
    const { hostname } = new URL(url);
    const blocked = /^(localhost|127\.\d+\.\d+\.\d+|0\.0\.0\.0|169\.254\.\d+\.\d+|::1)$/i.test(hostname);
    if (blocked) return res.status(400).json({ error: 'URL not allowed' });
} catch {
    return res.status(400).json({ error: 'Invalid URL' });
}
```

**`proxy-image.ts`**: 기존 `new URL(finalUrl)` 직후에 추가. URL repair 로직(double-https 수정 등)을 거친 `finalUrl` 기준으로 검사.
