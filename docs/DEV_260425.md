## 기존 로직 영향 검토

### Vercel 함수 타임아웃 — 주의 필요

`vercel.json` `maxDuration: 60` 설정 기준, Gemini 2.5 Flash ~200토큰/초 가정:

| 응답 길이 | 생성 시간 | 60s 타임아웃 |
|-----------|---------|------------|
| 8,192 tokens | ~41초 | 안전 ✅ |
| 16,000 tokens | ~80초 | **초과 ❌** |
| 32,768 tokens | ~164초 | **초과 ❌** |

16,000토큰(~13,000자) 이상 응답은 스트림 도중 강제 종료 가능.
일반적인 응답 범위(~5,000토큰)에서는 문제 없음.
코드 대량 생성 등 극단적 케이스에서만 해당. `maxDuration` 상향 시 완전 해소.

### 프론트엔드 — 영향 없음

- `setSessions` 청크별 호출 구조는 기존과 동일, 토큰 수 증가가 빈도에 영향 없음
- `react-markdown` 렌더링: 매우 긴 응답(30,000토큰+)에서 느려질 수 있으나 실용 범위 밖
- Supabase `TEXT` 타입 제한 없음 — DB 저장 영향 없음

### LangChain 경로 (drug_id, drug_info) — 영향 없음

약품 응답은 원래 짧아 실질적 변화 없음.
