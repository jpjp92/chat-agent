## 5. 응답 속도 최적화 (Performance)

### 5-1. YouTube fetch 병렬화 — `api/fetch-url.ts`

**문제**: OEmbed(최대 8초) → Page fetch(최대 10초) 직렬 실행으로 최악 **18초** 소요.  
두 요청은 완전 독립적이므로 병렬화 가능.

**수정**: `Promise.allSettled` 사용 — 한쪽 실패 시 다른 쪽 결과 유지, 독립 폴백 동작 보장.  
`clearTimeout`은 `allSettled` 완료 후 일괄 호출.

```ts
// Before: 직렬 (최악 18초)
const oRes = await fetch(oembed...);   // 최대 8초 대기
const pageRes = await fetch(page...);  // 그 다음 최대 10초 대기

// After: 병렬 (최악 10초)
const [oembedResult, pageResult] = await Promise.allSettled([
    fetch(oembed..., { signal: oembedController.signal }),
    fetch(page...,   { signal: pageController.signal }),
]);
clearTimeout(oembedTimeout);
clearTimeout(pageTimeout);
```

**Promise.all 대신 allSettled 선택 이유:**  
`Promise.all`은 하나라도 reject 시 전체 throw → 기존의 "한쪽 실패해도 빈 값으로 폴백" 동작과 달라짐.  
`allSettled`는 각 결과를 `{ status, value | reason }` 으로 반환하므로 독립 에러 처리 유지.

**효과**: 최악 케이스 18초 → 10초 (약 8초 단축). OEmbed가 빠른 일반 케이스에서는 체감 차이 없음.

## 6. PubChem SMILES 검증 통합 — `components/ChemicalRenderer.tsx`

### 6-1. 배경 및 문제

LLM이 생성한 SMILES는 단순 소분자(아스피린, 카페인)는 정확하지만 복잡한 분자(데스모프레신 등 매크로사이클릭 펩타이드)에서 부정확할 수 있음. PubChem(NIH 공개 API, 인증 불필요)으로 검증해 정확도 향상.

| 분자 유형 | LLM SMILES 신뢰도 | 처리 |
|-----------|------------------|------|
| 단순 소분자 (아스피린, 이부프로펜) | 높음 | PubChem과 동일 SMILES 반환 → 뱃지만 "PubChem 검증"으로 전환 |
| 복잡 펩타이드 (데스모프레신) | 낮음 (스테레오 누락, 원자 오류 가능) | PubChem canonical SMILES로 교체 |

---

### 6-2. 2단계 PubChem 조회 로직

```ts
const PUBCHEM_NAME_URL = 'https://pubchem.ncbi.nlm.nih.gov/rest/pug/compound/name';
const PUBCHEM_SMILES_URL = 'https://pubchem.ncbi.nlm.nih.gov/rest/pug/compound/smiles';

async function fetchPubChemSmiles(drugName: string, fallbackSmiles?: string): Promise<string | null> {
  // Step 1: 이름 기반 GET 조회
  // 한국어 혼합명에서 영어 추출: "아스피린 (Aspirin)" → "Aspirin"
  const engMatch = drugName.match(/\(([A-Za-z][^)]+)\)/);
  const queryName = engMatch?.[1]?.trim() ?? drugName;
  try {
    const res = await fetch(`${PUBCHEM_NAME_URL}/${encodeURIComponent(queryName)}/property/SMILES,MolecularFormula/JSON`,
      { signal: AbortSignal.timeout(5000) });
    if (res.ok) { ... return smiles; }
  } catch {}

  // Step 2: 이름 조회 실패 시 LLM SMILES 구조 POST 검증
  if (!fallbackSmiles) return null;
  const res = await fetch(PUBCHEM_SMILES_URL + '/property/SMILES,MolecularFormula/JSON', {
    method: 'POST',
    headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
    body: `smiles=${encodeURIComponent(fallbackSmiles)}`,
    signal: AbortSignal.timeout(6000),
  });
  return data?.PropertyTable?.Properties?.[0]?.SMILES ?? null;
}
```

**Step 1 실패 케이스**: 순한국어명(`히스타민`, `데스모프레신`) → PubChem 404 → Step 2로 폴백  
**Step 2 동작**: LLM SMILES를 PubChem에 POST해 유효 구조면 canonical SMILES 반환, 무효 구조면 400 반환
