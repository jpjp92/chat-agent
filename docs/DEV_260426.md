# DEV_260426 — 스트리밍 중 `**` 볼드 마커 미완성 렌더링 수정

## 배경

AI 응답에서 `**텍스트**` 볼드 강조가 가끔 `**` 기호 그대로 표기되는 문제 발생.

---

## 원인 분석

### 렌더링 구조

`renderContent()` → 텍스트를 viz 블록 기준으로 분할 → 각 텍스트 파트를 독립 `ReactMarkdown` 인스턴스로 렌더링.

### 원인별 분류

| # | 원인 | 빈도 | 수정 가능 여부 |
|---|------|------|--------------|
| 1 | 스트리밍 중 `**` 쌍 미완성 — 닫히는 `**`가 아직 미도착 | **가장 흔함** | ✅ 코드 수정 |
| 2 | viz 블록 기준 텍스트 분할 시 `**` 쌍이 파트 경계로 잘림 | 드묾 | △ 코드 수정 가능 (복잡) |
| 3 | 모델이 ` ** 텍스트 ** ` (앞뒤 공백 포함) 출력 — CommonMark 스펙상 볼드 불인식 | 간헐적 | △ 시스템 프롬프트로 완화 |
| 4 | `maxOutputTokens` 초과로 응답 잘림 — 닫히는 `**` 미도착 | 개선됨 | ✅ 32768으로 이미 상향 |

### 시스템 프롬프트가 최선이 아닌 이유

가장 빈번한 원인(스트리밍 중 쌍 미완성)은 모델 지시로 해결 불가 — 스트리밍 특성 자체의 문제.

### 기존 코드와의 관계

`ChatMessage.tsx`에 이미 동일한 패턴의 처리가 있음:

```ts
// 기존: dangling ``` 처리
const backticks = processedRemaining.match(/```/g);
if (backticks && backticks.length % 2 !== 0) {
    processedRemaining += '\n```';
}
```

`**` 에 대한 동일 처리가 누락된 상태였음.

---

## 수정 내용

**파일:** `components/ChatMessage.tsx`

두 곳(incomplete viz 분기 / normal 분기) 모두에 추가.

```diff
  // Safely close dangling code blocks during streaming
  const backticks = processedRemaining.match(/```/g);
  if (backticks && backticks.length % 2 !== 0) {
    processedRemaining += '\n```';
  }
+ // Close unclosed bold markers during streaming
+ if ((processedRemaining.match(/\*\*/g) || []).length % 2 !== 0) {
+   processedRemaining += '**';
+ }
```

### 동작 방식

스트리밍 도중 `**`의 총 개수가 홀수이면 문자열 끝에 `**`를 추가 → ReactMarkdown이 완전한 쌍으로 파싱 → 볼드 렌더링.
다음 청크 도착 시 실제 닫히는 `**`가 붙으면 appended `**`는 자연스럽게 중화됨.

### 미해결 케이스

- ` ** 텍스트 ** ` (공백 포함): CommonMark 스펙상 볼드 불인식. 빈도 높으면 시스템 프롬프트에 `**텍스트**` 형식(앞뒤 공백 없이) 명시 추가 검토.
- viz 블록 경계 분할: 모델이 `**` 안에 viz 블록을 생성하는 극히 드문 케이스. 현재 감수.

## 수정 2: MFDS 미등재 약품 웹 검색 폴백 개선

### 문제

"제놀한방파스"처럼 파스(패치), 연고, 액제 등 알약·정제가 아닌 의약품은 MFDS 알약식별 DB에 없음.
기존 코드는 MFDS 실패 시 DuckDuckGo 웹 검색을 내부적으로 시도하고 있었으나, LLM이 웹 검색 결과를 무시하고 "찾을 수 없습니다"를 출력하는 문제 발생.

멀티턴에서 사용자가 직접 "검색해줘" 요청 시에는 `general` intent → Google Search grounding → 정보 정상 출력.

### 원인

시스템 프롬프트와 툴 반환 지시의 충돌:
- **시스템 프롬프트**: `"MUST generate json:drug block for ANY drug info request"` (강한 지시)
- **툴 반환**: `"마크다운으로 안내하세요"` (약한 지시)

LLM이 `json:drug` 블록을 생성할 수 없지만(MFDS 데이터 없음) 시스템 프롬프트가 강제하므로, 양쪽 지시를 모두 만족할 수 없어 포기 → "찾을 수 없습니다" 출력.

### 수정 내용

**파일 1:** `api/_lib/agent/drug-info-tool.ts`

MFDS 실패 시 툴 반환 지시 강화 — 웹 검색 성공/실패 케이스 분리 및 `json:drug` 생성 명시 금지:

- `hasWebContent` 판별 후 명시적으로 `json:drug 블록을 생성하지 마세요` 지시 추가
- 웹 검색 결과 없을 때는 보유 지식 기반 안내로 분기

**파일 2:** `api/_lib/agent/prompt.ts`

- `PROACTIVE DRUG VISUALIZATION` 규칙에 `[MFDS_NOT_FOUND]` 예외 명시 (파스·연고 등 비알약 제형)
- `PRIORITY RULE`에도 MFDS_NOT_FOUND 예외 추가
- `drug_info` intent focus hint에 `CRITICAL EXCEPTION` 명시

**파일 3:** `api/chat.ts`

`on_chat_model_stream` 스트리밍 청크 sanitizer에 누출 방어 추가 (기존 K5 필터는 `on_chain_end` 경로만 커버):

```ts
sanitizedText = sanitizedText.replace(/`?json:drug`?\s*블록은\s*생성(?:하지\s*마세요|할\s*수\s*없습니다)[.]?\s*/g, '');
sanitizedText = sanitizedText.replace(/\[MFDS_NOT_FOUND\][^\n]*/g, '');
sanitizedText = sanitizedText.replace(/⚠️\s*CRITICAL INSTRUCTION:[^\n]*/g, '');
```

### 구조적 한계

파스·연고·시럽 등 비알약 제형은 MFDS 알약식별 DB에 원천 미등재 → DrugRenderer 카드 없이 마크다운으로만 안내. DuckDuckGo 파싱 실패 시 보유 지식 기반 답변(주의 안내 포함).
