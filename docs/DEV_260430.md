# YouTube 요약 처리 성능 개선 (2026-04-30 — 완료)

## 배경

YouTube URL 요약/분석 요청 시 응답까지 체감 대기 시간이 길다는 문제 제기.  
전체 요청 파이프라인을 검토한 결과 4개 병목 지점 확인 → 3개 파일 수정으로 개선.

---

## 파이프라인 구조 (개선 전)

```
[클라이언트]
  ① fetchUrlContent(url)         → 메타데이터 (직렬)
  ② fetchYoutubeTranscript(id)   → 자막 fetch (직렬, 클라이언트 timeout: 45s)

[서버 /api/fetch-transcript]
  ③ YouTube 페이지 HTML fetch    (timeout: 12s, 직렬)
  ④ caption XML fetch            (timeout: 15s, 직렬)
    → 자막 실패 시 최대 27s 소비 후 null 반환

[서버 /api/chat → LangGraph]
  ⑤ Router: gemini-2.5-flash-lite LLM 호출 (intent 분류, ~1~3s)
  ⑥ Generator: Gemini 요약 스트리밍
```

---

## 변경 1 — `src/hooks/useChatStream.ts` : 병렬 fetch

```ts
// Before: 직렬 (두 요청 시간의 합)
const metadata = await fetchUrlContent(url);
const transcript = await fetchYoutubeTranscript(videoId);

// After: 병렬 (가장 느린 쪽의 시간만)
const [metadata, transcript] = await Promise.all([
  fetchUrlContent(url),
  videoId ? fetchYoutubeTranscript(videoId) : Promise.resolve(null),
]);
```

- `fetchUrlContent`(메타데이터)와 `fetchYoutubeTranscript`(자막)는 순서 의존성 없음
- 예상 절감: **2~5s**

---

## 변경 2 — `api/_lib/agent/nodes/router.ts` : YouTube fast-path

```ts
// YouTube URL 감지 시 LLM 분류 없이 즉시 general 라우팅
const hasYoutubeUrl = /(?:youtube\.com\/|youtu\.be\/)/.test(textContent) ||
    (state.webContent && /(?:youtube\.com\/|youtu\.be\/)/.test(state.webContent));
if (hasYoutubeUrl && !hasMedicalKeyword) {
    console.log('[LangGraph] Router fast-path: YouTube URL → general');
    return { nextNode: "generator", intent: "general" };
}
```

- YouTube 요청은 항상 `general` intent → `gemini-2.5-flash-lite` 호출 불필요
- 기존 이미지 fast-path(`hasImage && !hasMedicalKeyword`)와 동일한 패턴으로 추가
- 의약품 키워드 동반 시 fast-path 제외 (기존 LLM 분류 유지)
- 예상 절감: **1~3s**

## 변경 3 — `api/fetch-transcript.ts` : 서버 timeout 단축

| 구간 | Before | After |
|------|--------|-------|
| YouTube 페이지 HTML fetch | 12s | 6s |
| caption XML fetch | 15s | 8s |
| **실패 시 최대 대기 합산** | **27s** | **14s** |

자막이 없는 영상, 자막 URL 접근 불가, YouTube 서버 지연 등 실패 케이스에서 사용자 대기 시간 단축.

## 케이스별 효과 요약

| 케이스 | 개선 전 | 개선 후 |
|--------|---------|---------|
| 자막 추출 **성공** (일반적인 경우) | 메타데이터 + 자막 시간의 합 | 둘 중 느린 쪽 시간만 |
| 자막 추출 **실패** (timeout 도달) | 최대 ~27s 대기 후 fallback | 최대 ~14s 대기 후 fallback |
| Router LLM 호출 | 매 요청마다 ~1~3s | 0s (즉시 라우팅) |

---

## 미개선 항목 (구조적 한계)

- **native video fallback 지연**: 자막 없을 때 Gemini가 영상 직접 다운로드+분석 — 영상 길이에 따라 30~60s 이상 소요. 구조상 제거 불가, 사용자 안내 메시지로 대응 중.
- **트랜스크립트 캐싱 없음**: 동일 영상 반복 요청 시 매번 re-fetch. 세션 내 `lastActiveDoc.extractedText`로 부분 완화되어 있음.
