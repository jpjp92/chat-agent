## 3. 세션 초기화 "연결에 실패했습니다" 에러 화면 수정

### 원인 분석

**경로 1 — `initAuth().catch()` 핸들러** (`useAuthSession.ts`)

`initAuth()` 내부에서 예외 발생 시 catch 블록이 `setIsAuthLoading(false)`만 호출하고 `setCurrentUser`를 명시 호출하지 않아, `isAuthLoading=false + currentUser=null` 조합으로 에러 화면 표시.

**경로 2 — localStorage 파싱 실패 후 `else` 미진입** (`useAuthSession.ts`)

`JSON.parse(savedUser)` 실패 시 `catch` 블록에서 localStorage만 제거하고 신규 게스트 생성(`else` 분기)으로 진입하지 않아 `user=null`인 채로 진행.

**경로 3 — `handleReset` React 상태 선행 업데이트** (`App.tsx`) ← 실제 재현 경로

```ts
// Before: clearStoredUser()가 setCurrentUser(null) 호출 → reload 전에 리렌더 발생
const handleReset = () => {
    clearStoredUser();           // → setCurrentUser(null), isAuthLoading=false 그대로
    window.location.reload();    // 리렌더 후 실행
};
// 결과: isAuthLoading=false + currentUser=null → 에러 화면 0.1초 순간 표시
```

### 수정 내용

**`App.tsx` — `handleReset` React 상태 업데이트 제거**

```ts
// After: React 상태 변경 없이 localStorage만 직접 제거 후 reload
```

- `.catch()` 핸들러에 `setCurrentUser(null)` 명시 추가

## 5. ConstellationRenderer 렌더링 실패 수정

### 증상
`물병자리 시각화해줘` 질의 시 AI가 `"lines": [, , , ]` (sparse array) 생성 → `JSON.parse` 실패 → catch에서 raw JSON이 채팅에 그대로 노출.

### 수정 1 — `ChatMessage.tsx` JSON repair + 조용한 에러 처리

```ts
// parse 전 repair: sparse array·trailing comma 제거
const repaired = match[2]
    .replace(/,\s*(?=[,\]\}])/g, '')
    .replace(/\[\s*,/g, '[')
    .replace(/,\s*\]/g, ']');
const jsonData = JSON.parse(repaired);

// catch: raw JSON 노출 대신 조용히 skip
} catch (e) {
    // 유효하지 않은 viz 블록은 무시 — 텍스트 설명은 그대로 표시됨
}
```

### 수정 2 — `prompt.ts` CRITICAL JSON RULES 추가

- `lines` 배열에 빈 슬롯·sparse array 절대 금지 명시
- 정확한 IAU lines 미확인 시 순차 연결(`[[0,1],[1,2],...]`)로 대체하도록 지시
- 물병자리 lines 예시 직접 추가: `[[1,0],[0,2],[2,3],[1,4]]`

### 복잡 분자 겹침 수정 (리팜피신 등)

리팜피신(`CC1...` SMILES length 179) 같은 복잡한 고리 구조에서 OH기·CH₃기가 겹치던 문제 해결.

```ts
// Before → After
explicitHydrogens: true  → false   // 명시 수소 제거 — CH3·OH 노드 클러터 제거 (핵심)
overlapResolutionIterations: 2  → 8  // 레이아웃 반복 증가 → 겹침 해소 시도 강화
bondLength: 35.0 → 37.0             // 결합 간격 소폭 확대 (bondSpacing도 비례 조정)
bondSpacing: 0.18 * 35.0 → 0.18 * 37.0
```

### 적응형 높이 (`resolvedHeight`)

`height` prop이 없을 때 SMILES 길이 기반으로 자동 결정.

```ts
const resolvedHeight = height ?? (smiles.length > 120 ? 460 : smiles.length > 60 ? 380 : 300);
```

| SMILES 길이 | 높이 | 대상 |
|-------------|------|------|
| > 120 | 460px | 리팜피신급 복잡 분자 |
| 60–120 | 380px | 중간 복잡도 |
| < 60 | 300px | 단순 분자 |

### 버그 수정 — `height` 참조 오류

`resolvedHeight` 도입 후 두 곳이 여전히 `height` prop을 참조해 `undefined` → 렌더 왜곡 발생.

```tsx
// SVG viewBox
// Before: viewBox={`0 0 ${width} ${height}`}
// After:  viewBox={`0 0 ${width} ${resolvedHeight}`}

// 다운로드용 tempSvg
// Before: tempSvg.setAttribute("height", height.toString())
// After:  tempSvg.setAttribute("height", resolvedHeight.toString())
```
