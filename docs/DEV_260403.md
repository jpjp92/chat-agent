# 타이레놀 검색 문제 분석 (2026-04-03)

## 문제점
- 이미지는 정확하게 `타이레놀정500밀리그람(아세트아미노펜)`의 데이터를 캐시하고 있음 (각인: TYLENOL/500)
- 하지만 **pharm.or.kr URL을 가져올 때** 약품명만으로 검색해서 다른 타이레놀 제품(우먼스타이레놀 등)과 매치될 수 있음

## 원인 분석
`getPharmOrKrDetailUrl()`이 MFDS에서 추출한 **각인 정보를 사용하지 않고** 약품명만으로 검색하기 때문입니다.
- 점수 계산에서 용량은 고려하지만 각인은 검증하지 않음
- 약품명만으로는 유사한 제품들과 구분이 어려움

## 해결책
MFDS 데이터에서 추출한 각인(imprint) 정보도 `getPharmOrKrDetailUrl()`에 전달해서, 검색 결과를 각인으로 추가 검증합니다.

### 수정 대상 파일
- [api/_lib/agent/drug-info-tool.ts](file:///home/jpjp92/devs/github/chat-agent/api/_lib/agent/drug-info-tool.ts)
  - L75-130: `getPharmOrKrDetailUrl()` 함수 - 각인 파라미터 추가 및 검증 로직 추가
  - L160-230: 함수 호출부 - 각인 정보 전달

### 구현 계획
1. `getPharmOrKrDetailUrl()` 함수에 `imprint` 파라미터 추가
2. 검색 결과 필터링 시 각인 정보로 추가 검증
3. 함수 호출 부분에서 MFDS 데이터의 각인 정보 전달

## 구현 완료 (2026-04-03)

### 수정 내용
1. **함수 서명 (L71)**: 
   - `async function getPharmOrKrDetailUrl(drugName: string, imprint?: string)`
   - `imprint` 파라미터를 선택사항으로 추가

2. **각인 정규화 로직 (L125-130)**:
   - 전달받은 imprint를 정규화 (`-`, 공백 제거, 대문자 변환)
   - 각인이 제공되면 로깅

3. **검색 결과 필터링 로직 (L162-175)**:
   - `extractImprintFromHtml()` 헬퍼 함수 추가
   - 각인 검증이 활성화되면 검색 결과를 각인으로 필터링
   - imprint 미스매치 항목은 제외

4. **폴백 검색 개선 (L319-324)**:
   - MFDS 첫 번째 결과의 각인(앞면 또는 뒷면)을 추출
   - pharm.or.kr 폴백 검색에서 각인 정보와 함께 검색
   - `getPharmOrKrDetailUrl(baseName, imprintForValidation)` 호출

### 효과
- 여러 타이레놀 제품(우먼스타이레놀 등)이 있을 때 각인으로 정확한 제품 구분
- 용량 + 각인 이중 검증으로 오매칭 방지
- MFDS에서 추출한 정확한 각인으로 pharm.or.kr 데이터 검증
